# MongoDB Integration

## Overview

The MongoDB integration enables Tracer to diagnose and investigate issues in MongoDB deployments. It provides diagnostics tools to query MongoDB server status, identify slow queries, check replica set health, and analyze connection patterns.

## Capabilities

Tracer MongoDB integration includes these capabilities:

- **Server Health Monitoring** - Query server status, connection counts, memory usage, and operation counters
- **Current Operations Analysis** - Identify long-running operations and locks
- **Replica Set Status** - Monitor replica set members, sync lag, and quorum health
- **Slow Query Profiling** - Analyze slow queries from the system.profile collection
- **Collection Statistics** - Check collection size, index statistics, and data distribution

## Prerequisites

- **MongoDB Instance** - MongoDB 4.0+ (4.4+ recommended)
- **Network Access** - Connection string must be reachable from Tracer environment
- **Credentials** - Valid authentication credentials (if auth enabled)
- **Python Package** - `pymongo>=4.6.0` (included in Tracer dependencies)

## Setup

### 1. Via CLI Interactive Setup

```bash
opensre integrations setup
```

Select `mongodb` when prompted:

```
? Select integration to configure:
  > mongodb
    datadog
    grafana
    ...
```

You'll be prompted for:

- **Connection String** - MongoDB connection URI (e.g., `mongodb+srv://user:pass@cluster.example.net`)
- **Database** - Target database name (optional, used for profiler and stats queries)

### 2. Via Environment Variables

Add to `.env`:

```bash
MONGODB_CONNECTION_STRING=mongodb+srv://user:pass@cluster.example.net
MONGODB_DATABASE=production
MONGODB_AUTH_SOURCE=admin
MONGODB_TLS=true
```

**Connection String Format:**

- **Localhost** - `mongodb://localhost:27017`
- **Replica Set** - `mongodb://host1,host2,host3/?replicaSet=rs0`
- **Atlas (SRV)** - `mongodb+srv://user:pass@cluster.example.net`
- **Username/Password** - `mongodb://user:pass@host:27017/database`

**Environment Variables:**

| Variable                    | Default | Description                          |
| --------------------------- | ------- | ------------------------------------ |
| `MONGODB_CONNECTION_STRING` | -       | **Required.** Connection URI         |
| `MONGODB_DATABASE`          | (empty) | Target database for profiler & stats |
| `MONGODB_AUTH_SOURCE`       | `admin` | Authentication database              |
| `MONGODB_TLS`               | `true`  | Use TLS for connection               |

### 3. Via Store (Persistent)

Integrations are automatically persisted to `~/.tracer/integrations.json`:

```json
{
    "version": 1,
    "integrations": [
        {
            "id": "mongodb-prod",
            "service": "mongodb",
            "status": "active",
            "credentials": {
                "connection_string": "mongodb+srv://user:pass@cluster.example.net",
                "database": "production",
                "auth_source": "admin",
                "tls": true
            }
        }
    ]
}
```

## Configuration

### Connection String

MongoDB connection strings support:

**Standard URI format:**

```
mongodb://[username[:password]@]host1[:port1][,host2[:port2],...][/?options]
```

**Examples:**

```
# Localhost (no auth)
mongodb://localhost:27017

# Username/password auth
mongodb://user:password@host:27017/database?authSource=admin

# Replica set
mongodb://host1:27017,host2:27017,host3:27017/?replicaSet=rs0

# Atlas (SRV)
your - mongodb+srv://user:password..... link

# Atlas with specific database and auth
your - mongodb+srv://user:password..... link
```

**Special Characters in Credentials:**

Use URL encoding for special characters in username or password:

```
@ → %40
: → %3A
# → %23
```

### TLS/SSL Configuration

Set `MONGODB_TLS=true` (default) to use TLS:

```bash
# Use TLS (recommended for production)
MONGODB_TLS=true

# Disable TLS (localhost only)
MONGODB_TLS=false
```

For certificate validation issues, use environment variables:

```bash
# Custom CA certificate (PEM format)
MONGODB_CA_CERT=/path/to/ca.pem

# Skip certificate validation (not recommended)
MONGODB_TLS_INSECURE=true
```

### Database Selection

Specify the target database for queries:

```bash
MONGODB_DATABASE=production
```

This database is used for:

- **Profiler queries** - Reads from `system.profile` collection
- **Collection stats** - Retrieves statistics for collections in this database
- **Database stats** - Gets overall database metrics

If not configured, profiler and collection queries will fail with an error.

## Usage in Investigations

### Alert Recognition

Tracer automatically detects MongoDB-related alerts via:

- **Alert Annotations** - Fields like `mongodb_database`, `mongodb_collection`, `mongodb_instance`
- **Alert Labels** - `alertname` containing "mongodb" or similar
- **Configured Integration** - If MongoDB is configured, it's available for investigation

### Available Tools

Five MongoDB tools are available during investigations:

#### 1. **MongoDB Server Status**

```
Tool: query_mongodb_server_status
Input:
  - connection_string (required)
  - auth_source (default: admin)
  - tls (default: true)

Output:
  - version
  - uptime_seconds
  - connections (current, available, total_created)
  - opcounters (insert, query, update, delete, etc.)
  - memory (resident_mb, virtual_mb)
  - storage_engine
```

**Use Cases:**

- Identify high connection counts
- Check memory usage and resident set
- Track operation volume trends

#### 2. **MongoDB Current Ops**

```
Tool: query_mongodb_current_ops
Input:
  - connection_string (required)
  - auth_source (default: admin)
  - tls (default: true)
  - threshold_ms (default: 1000)

Output:
  - threshold_ms
  - total_ops
  - returned_ops
  - operations[] (opid, op, ns, secs_running, wait_for_lock, plan_summary)
```

**Use Cases:**

- Find long-running queries
- Identify lock contention
- Discover slow write operations

#### 3. **MongoDB Replica Status**

```
Tool: query_mongodb_replica_status
Input:
  - connection_string (required)
  - auth_source (default: admin)
  - tls (default: true)

Output:
  - set_name
  - my_state
  - heartbeat_interval_ms
  - members[] (name, state, health, uptime_seconds, optime, ping_ms)
```

**Use Cases:**

- Check replica set quorum health
- Identify unreachable members
- Monitor optime lag between nodes

#### 4. **MongoDB Profiler**

```
Tool: query_mongodb_profiler
Requires: MONGODB_DATABASE configured

Input:
  - connection_string (required)
  - auth_source (default: admin)
  - tls (default: true)
  - threshold_ms (default: 100)

Output:
  - profiling_level (0, 1, or 2)
  - threshold_ms
  - total_entries
  - entries[] (op, ns, millis, plan_summary, docs_examined, keys_examined)

Note: Requires profiling enabled on the database
```

**Use Cases:**

- Identify slow queries
- Analyze index usage (docs_examined vs keys_examined)
- Find query plans with full collection scans

#### 5. **MongoDB Collection Stats**

```
Tool: query_mongodb_collection_stats
Requires: MONGODB_DATABASE configured

Input:
  - connection_string (required)
  - collection (required)
  - auth_source (default: admin)
  - tls (default: true)

Output:
  - ns
  - count
  - size_bytes
  - avg_obj_size_bytes
  - storage_size_bytes
  - total_index_size_bytes
  - index_count
  - capped
```

**Use Cases:**

- Check collection size and growth
- Identify bloated collections
- Analyze index effectiveness

## Troubleshooting

### Connection Refused

**Symptom:** "MongoDB connection failed: Connection refused"

**Solutions:**

- Verify connection string is correct
- Check MongoDB is running on the specified host/port
- Verify network connectivity (firewall rules, security groups)
- For Atlas: ensure IP whitelist includes Tracer's IP

### Authentication Failed

**Symptom:** "MongoDB connection failed: authentication failed"

**Solutions:**

- Verify username and password are correct
- Check `authSource` matches the database where credentials exist (usually `admin`)
- Ensure user has admin role or appropriate permissions
- Try connecting manually: `mongosh "mongodb://..." --username user`

### TLS Certificate Error

**Symptom:** "SSL: CERTIFICATE_VERIFY_FAILED"

**Solutions:**

- For self-signed certs: set `MONGODB_TLS_INSECURE=true` (dev only)
- Provide CA cert: `MONGODB_CA_CERT=/path/to/ca.pem`
- For Atlas: ensure connection string uses `mongodb+srv://` protocol

### Profiler Not Available

**Symptom:** "Profiling is disabled on this database"

**Solutions:**

- Enable profiler on the database:
    ```javascript
    db.setProfilingLevel(1); // Log slow queries (>100ms)
    db.setProfilingLevel(2); // Log all queries
    ```
- Or configure threshold in tool parameters

### Replica Set Not Configured

**Symptom:** "Server is not part of a replica set"

**Information:**

- This is expected for standalone MongoDB instances
- Replica set tools will return empty member list
- Server status and profiler tools still work

## Security Best Practices

1. **Use Strong Credentials**
    - Avoid default/simple passwords
    - Use long random passwords for production

2. **Restrict Access**
    - Use read-only users for monitoring
    - Minimize required privileges
    - Don't use admin credentials if monitoring-only access needed

3. **Use TLS/SSL**
    - Enable `MONGODB_TLS=true` in production
    - Use certificate validation
    - Never skip certificate checks in production

4. **Secure Connection Strings**
    - Store in `.env` (not in code)
    - Rotate credentials periodically
    - Audit who has access to integration config

5. **Network Security**
    - Use VPN/private networks when possible
    - Restrict MongoDB port access to known IPs
    - Monitor for unusual connection patterns

## Performance Considerations

### Query Impact

MongoDB diagnostic tools use read-only commands:

- `serverStatus` - Quick, minimal I/O
- `currentOp` - Quick, scans in-memory operations
- `replSetGetStatus` - Instant, no query execution
- `system.profile` queries - Depends on profile collection size
- `collStats` - Can be slow on very large collections

**Recommendations:**

- Run profiler queries on secondaries when possible
- Use reasonable `threshold_ms` values (default 100-1000ms)
- Avoid collection stats on massive collections during peak hours

### Connection Pooling

By default, Tracer uses:

- Timeout: 5 seconds
- Max connections: 1
- Close after each query

For frequent queries, consider increasing connection pool size in `MONGODB_CONNECTION_STRING`.

## FAQ

**Q: Can I use MongoDB for alerts?**
A: Not yet - MongoDB integration is for investigation diagnostics only. Configure alert sources in your monitoring platform (Datadog, Grafana, etc.).

**Q: Does this tool modify my MongoDB?**
A: No. All operations are read-only. No data is written or modified.

**Q: Does this support MongoDB Atlas?**
A: Yes! Use the standard connection string format from Atlas UI.

**Q: Can I use multiple MongoDB instances?**
A: Currently, only one MongoDB integration is supported. You can change which instance by updating the connection string.

**Q: Does this work with MongoDB Sharded Clusters?**
A: Yes. Use the mongos server address or cluster connection string.
